HTML & the DOM Tree

// tokenization, semantic nodes, and the render path

01 The Architect

Sir Tim Berners-Lee

In 1991, Berners-Lee created HTML at CERN so researchers could share documents. By designing it as a declarative markup language rather than procedural, he ensured the web could be processed by any machine regardless of hardware architecture.

02 The Tokenization Algorithm

When a browser receives an HTML byte stream, it runs Tokenization — converting characters into tokens (StartTag, EndTag, Attribute, Comment). Unlike most languages, HTML has a built-in error recovery algorithm that auto-fixes missing or malformed tags. However, this costs CPU cycles and can cause a Flash of Unstyled Content (FOUC).

03 The DOM Tree

HTML is not the final state. The browser builds an internal structure called the DOM Tree — every tag becomes a Node. CSS is then applied to create the Render Tree. Only visible nodes (not display: none) are painted to screen. This distinction is critical for performance.

05 The Critical Rendering Path

HTML is render-blocking — the browser waits for the full DOM before painting. JavaScript is parser-blocking — hitting a <script> tag stops DOM construction entirely until the script downloads and executes. This is why serious apps use the defer attribute.
