01 The Architect

Brendan Eich

In 1995, Eich wrote the first JavaScript prototype for Netscape Navigator in just 10 days. His decision to include Prototypal Inheritance and First-Class Functions turned JS into one of the most flexible languages ever built — now the undisputed king of the internet.

02 V8 Engine & JIT Compilation

Modern browsers use the V8 Engine (Google) for Just-In-Time (JIT) Compilation. Instead of interpreting line-by-line, V8 compiles JS into highly optimized machine code while the program runs. A "Profiler" watches code and re-compiles hot paths to make them even faster.

03 The Single-Threaded Event Loop

JS is single-threaded — one command at a time. It handles concurrency via the Event Loop: heavy tasks go to Web APIs (background), then enter the Callback Queue. The Event Loop pushes the next task into the Call Stack only when it's empty. This is why JS is "non-blocking."

04 Execution Context & Hoisting

When JS runs, the engine creates a Global Execution Context in two phases:

  1. Memory Creation: Scans for variables/functions and reserves memory (Hoisting).
  2. Execution: Assigns values and runs the code.

This is why var is undefined before declaration, but let/const throw a Reference Error (Temporal Dead Zone).

05 Prototypal Inheritance

JS doesn't have "true" classes — the class keyword is syntactic sugar. Every object has a hidden link to its Prototype. If a property isn't found, JS "climbs the chain" to find it. This is called the Prototype Chain.
