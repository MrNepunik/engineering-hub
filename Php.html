01 The Architect

Rasmus Lerdorf

Created in 1994, PHP began as CGI scripts to track visits to Lerdorf's online resume. He never intended it to be a full language — but its ability to embed directly into HTML made it explode in popularity. Today PHP powers 75%+ of the web, including WordPress and Wikipedia.

02 The Request-Response Cycle

PHP is strictly server-side. The browser never sees PHP code. When a user requests a .php file, the web server (Apache/Nginx) passes it to the PHP Interpreter, which executes the logic, generates a plain HTML string, and sends that back to the client. This is how PHP hides database credentials and business logic.

03 The Zend Engine

The Zend Engine runs PHP in 4 stages:

  1. Scanning: PHP code → Tokens
  2. Parsing: Tokens → Abstract Syntax Tree (AST)
  3. Compilation: AST → Opcodes
  4. Execution: Opcodes are run and output is returned

OPcache stores compiled Opcodes in RAM so the engine skips re-parsing on every page load.

04 Typed Properties & Memory

Modern PHP (8.x+) supports Strict Typing. Forcing variables to specific types (int, string, bool) leads to better memory management and fewer runtime errors — a major improvement over the loosely typed PHP of old.

05 Database Integration (PDO)

PHP's greatest strength is native database connectivity. PDO (PHP Data Objects) is an abstraction layer that protects against SQL Injection via "prepared statements" — ensuring user input can never corrupt database commands.
